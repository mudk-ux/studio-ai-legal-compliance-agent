Three Implementations, One Platform

A code-level review of the Studio Compliance platform across its three generations: the original single-agent baseline, the ADK 2.0 multi-agent studio (both on github.com/mudk-ux/studio-ai-legal-compliance-agent, branch main), and the production MVP 1.0 rewrite (branch v2, tag mvp-1.0) — what each accomplished, where each fell short, and what remains.

Generation 1

Original Baseline — Single-Agent Coordinator

Track 1 of the original repository · src/agent.py, src/tools/gcp_ml_tools.py, deploy/agent.py, frontend/app.py · deployed to Vertex AI Reasoning Engine · ~35 s per audit

The baseline established the domain decomposition that survived all three generations: screenplays, set/wardrobe images, and rough-cut video vetted against sponsor-exclusivity deals, Standards & Practices rules, and right-of-publicity exposure. It shipped a real ADK agent to a real cloud runtime — with a perception layer that quietly substituted simulation when the real thing failed.

Accomplished

  • The right problem, decomposed correctly — the modality split and rule taxonomy carried through every later generation.
  • A genuine ADK agent with tool bindings and instruction prompt, deployed live to Reasoning Engine and returning verdicts.
  • Real GCP API wrappers with native gs:// support in live mode.
  • Typed Pydantic report schemas; Streamlit operator dashboard; structured logging and PII-redaction utilities.

Limitations · simulated parts

  • Filename fallback was reachable in production. Any auth/quota failure silently downgraded detection to filename keywords with invented confidences.
  • NER fallback was a hardcoded list of the exact names in the sample scripts — high recall on the demo, none beyond it.
  • The "0/3-Plus Census rule" existed as a label on findings, not as an executed check; every detected person became a violation.
  • Script compaction kept the first 20% and last 75% of a capped window — the middle of a feature was never read.
  • Report metadata cited gemini-3.5-pro, a model that does not exist.

Drawbacks

  • Silent degradation is the worst failure mode for a legal tool: an unanalyzed asset could return CLEARED.
  • Verdicts extracted from chat text by substring search ("BLOCKED" in response).
  • Broken video timecodes made VFX slates unusable in editorial.
  • Hardcoded project IDs and buckets; committed .pyc files and ~100 MB of media; no unit tests of policy logic.

Generation 2

ADK 2.0 Multi-Agent Studio

Track 2 of the original repository · adk2_multiagent_studio/, src/orchestration.py · second Reasoning Engine deployment · ~0.8 s per audit

The second generation introduced the coordinator–specialist decomposition and the human-in-the-loop concept — both of which became real in MVP 1.0. In this generation they were structural sketches: the "agents" were plain Python classes carrying model-name strings, and no LLM was invoked anywhere in the graph. Its speed advantage over Generation 1 (~42×) was the speed of regex against an LLM round-trip.

Accomplished

  • The multi-agent decomposition — coordinator, script specialist, brand specialist, remediation compiler — that MVP 1.0 later implemented with real LLM agents.
  • The HITL contract design: pause payloads, reviewer roles, resume tokens — a good schema awaiting an implementation.
  • Modality routing and per-node execution traces.
  • A second live Reasoning Engine deployment alongside the baseline.

Limitations · simulated parts

  • No LLM in the loop. model_tier strings were decorative; specialists executed keyword matching over the same dual-mode tool outputs as Gen 1.
  • HITL never paused anything — findings were annotated and the workflow ran to completion.
  • The flagship module imported src.models.schemas, a path that did not exist; as committed, it could never have been imported successfully.
  • Perception inherited the filename-fallback layer wholesale.

Drawbacks

  • Architecture as nomenclature — class names promised behavior the code did not perform, making the system harder to evaluate honestly than a plain pipeline.
  • Benchmark claims were circular: the "40-cell matrix" scored a system recognizing its own test filenames, and the report's "100% consistency" summary was written unconditionally, even for errored runs.
  • The headline latency/accuracy comparison against Gen 1 measured different mechanisms, not better ones.
  • Near-duplicate source trees (src/, deploy/src/, adk2_multiagent_studio/src/) drifting independently.

Generation 3 — current

MVP 1.0 — Production Rewrite

Branch v2, tag mvp-1.0 · src/studio_compliance/ · two Agent Engine deployments (baseline + multi-agent) · 73 unit tests · verified across five QA rounds and a final acceptance run in an enterprise GCP environment

A ground-up rewrite around one principle: evidence over vibes. Both deployable tracks are real ADK LlmAgents, but they share a single audited pipeline — agents route, reason, and explain; they cannot invent facts. There is no fallback path anywhere in production code: a failed perception stage produces a FAILED verdict ("this asset was not fully vetted"), never a fabricated clearance. Mock perception exists only inside the test suite.

Accomplished · verified live

  • Every README claim is backed by a live corp run: deployment (~90 s builds), smoke gates, remote evals, specialist routing, constraint gating, HITL lifecycle.
  • Fail-loud semantics end to end — zero silent failures across all rounds; infra errors surface as FAILED, never as clearances.
  • Honest evaluation: labeled manifest, computed scores, unstable assets explicitly unscored, zero forbidden-entity (precision-trap) violations.
  • Deployable by anyone: env-driven config, Terraform (APIs, bucket TTLs, service-agent IAM), CI, no hardcoded identity.
  • Real HITL: persisted pending reviews, named-reviewer approve/enforce, double-resolution rejected, verdict recompute.

Limitations · known behavior

  • The two remaining verdict deltas are honest conservatism: an 1895 play returns CONDITIONAL because real institutions ("Tories", "Bankruptcy Court") appear in dialogue with no clearance on file.
  • 10-minute-plus video works locally (~9.4 min analysis) but hits corporate-proxy 503 stream resets when run through the deployed engine — an environment constraint, one retry built in.
  • Video Intelligence emits sub-second flicker detections on CG content — reported truthfully as MEDIUMs today.
  • LLM portrayal-risk grading (HIGH vs. MEDIUM for living figures) can vary between runs; documented in the manifest rather than masked.
  • Logo detection cannot see pattern-similarity infringement — the Damier-style lookalike bag legitimately clears under current rules.

Drawbacks · accepted trade-offs

  • Video Intelligence dominates cost and wall-clock time (per-minute billing; ~10 min for a 12-min 1080p asset).
  • Report and entity payloads live in GCS — callers need bucket access; the conversational reply alone is deliberately not the full record.
  • The file-per-record HITL store is safe for a small reviewer team, not for high-concurrency review desks.
  • Deterministic policy tables (aliases, substance brands) require curation as coverage grows.

The through-line: Generation 1 proved the domain and the deployment path. Generation 2 sketched the right architecture without implementing it. Generation 3 kept both contributions — the domain decomposition and the coordinator–specialist shape — and rebuilt the execution so that every number the system reports can be traced to a real API response, a real model call, or a named human decision.

After the MVP

Future Work

Deliberately deferred, in rough priority order. None block production use of MVP 1.0 for the verified workflows.

Segment-scoped video analysis

Analyze editor-specified ranges (video_context.segments) or poll long-running operations asynchronously. Removes the 10-minute synchronous stream that corporate proxies reset, and cuts Video Intelligence cost 5–10× on feature-length cuts.

Reliability · Cost

Detection quality floor for video

A track-duration / confidence threshold as an auditable policy rule, converting sub-second flicker false positives on CG content into advisory findings instead of MEDIUMs.

Precision

Script–video timestamp alignment

Speech-to-text plus forced alignment builds a scene index joining screenplay text to timecodes — enabling contextual checks logo detection cannot see: negative sponsor portrayal, behavioral S&P content, likeness appearances. Contextual LLM judgments would route to HITL, never auto-block.

Capability

Pattern-similarity infringement pass

A contextual multimodal review for trade-dress lookalikes (the Damier-style bag) where no explicit logo exists — the known gap in logo-model perception.

Capability

Period / historical-context handling

The last verdict deltas come from real institutions in period dialogue. Options: an era-aware clearance whitelist, or an LLM pass classifying organizational references as historical-context vs. commercial — keeping the conservative default.

Precision

Upload and transcode efficiency

Hash-based upload de-duplication and low-resolution proxy transcodes before Video Intelligence submission — the audit found proxies preserve logo legibility at a fraction of the bandwidth and processing cost.

Cost

HITL on Firestore

Transactional review records for concurrent legal reviewers, replacing the file-per-record GCS store once the reviewer team grows.

Scale

Hardening the agent contract

A response_schema on the coordinator and client-side 429/backoff retries in the perception layer — belt-and-suspenders over the already-robust report_uri retrieval path.

Robustness
Dataset hygiene, carried forward: the benchmark suite still contains two byte-identical file pairs (the "Elephants Dream teaser" is device-promo footage; the "beverage can" is the handbag image). Sourcing true negative-control media would sharpen the eval, and the ground-truth manifest is designed to absorb new labeled assets as one-line additions.
